# Incident Tracking Metrics

## Purpose

Measure incident handling performance and provide visibility into security operations activities.

## Metrics

| Metric | Description |
|----------|-------------|
| Total Incidents | Total number of security incidents created. |
| Incident Status | Number of Active, Closed, and Resolved incidents. |
| Incident Severity | Distribution of High, Medium, Low, and Informational incidents. |
| Average Resolution Time | Average time taken to investigate and resolve incidents. |

## Data Source

- Microsoft Sentinel
- Incident Management

## Objective

These metrics help evaluate incident response efficiency, monitor analyst workload, and identify opportunities to improve response processes.
