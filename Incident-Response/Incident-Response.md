# Incident Response

## Overview

This project follows a structured incident response process for security alerts generated by Microsoft Sentinel.

## Incident Response Workflow

### 1. Alert Analysis
Review the alert details, affected assets, severity, and associated analytics rule.

### 2. Timeline Review
Analyze the sequence of security events to understand attacker activity.

### 3. User Investigation
Identify the affected user account and review authentication activity.

### 4. Host Investigation
Examine the affected endpoint for suspicious processes, network connections, and system activity.

### 5. Process Analysis
Review process creation events and command-line execution using Sysmon telemetry.

### 6. Evidence Collection
Collect relevant Windows Security Events, Sysmon logs, and Sentinel investigation data.

### 7. Root Cause Analysis
Determine how the incident occurred and identify the initial attack vector.

### 8. Containment
Isolate affected systems or accounts to prevent further impact.

### 9. Recovery
Restore normal operations after verification and remove malicious activity.

### 10. Lessons Learned
Document findings and improve future detections, monitoring, and response procedures.
